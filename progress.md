## Goal and Objective
The goal of building this simple web application is to gain practical exposure to the development workflow and surface-level familiarity with the modern web application framework ecosystem.  
In other words, the goal of this project is not mastery, but to develop foundational understanding of how each technology fits into a real-world application.

2. **System Design**  
    Before starting to code and develop the application, I made:  
        - a Context Diagram  
        - a Data Flow Diagram  

    These diagrams are used to:  
        - Identify system boundaries  
        - Visualize user interactions  
        - Understand data movement between the frontend, backend, authentication layer, and database  
        - Clarify the responsibilities of each technology in the stack  

    This design step was done to reduce implementation errors, improve clarity, and ensure the application architecture aligns with real-world development practices.
